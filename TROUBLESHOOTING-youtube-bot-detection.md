# YouTube "Sign in to confirm you're not a bot" — yt-dlp

## Problem

yt-dlp fails to extract/download YouTube videos with this error:

```
ERROR: [youtube] fYT7969YIwU: Sign in to confirm you're not a bot.
Use --cookies-from-browser or --cookies for the authentication.
```

This happens even with valid, authenticated cookies. YouTube's bot detection now requires **two things**:
1. **Cookies** — proves identity (we have these)
2. **Proof of Origin (PO) tokens** — cryptographic proof the request comes from a real browser environment, generated by YouTube's BotGuard JavaScript

## Current Setup

- **yt-dlp version:** 2026.02.21 (latest)
- **Runtime:** Deno (installed in Dockerfile.app, replaced Node.js)
- **Cookies:** Netscape-format file at `/usr/src/app/cookies/youtube.com.txt` with full Google/YouTube auth cookies (LOGIN_INFO, SID, SSID, HSID, SAPISID, SIDCC, VISITOR_INFO1_LIVE, YSC, etc.)
- **PO token provider:** `brainicism/bgutil-ytdlp-pot-provider:latest` running as sidecar container on port 4416
- **Plugin:** `bgutil-ytdlp-pot-provider` pip package installed
- **Extractor args configured in `src/downloader.py`:**
  ```python
  "extractor_args": {
      "youtube": {"player_client": ["default"]},
      "youtubepot-bgutilhttp": {"base_url": ["http://pot-provider:4416"]},
  }
  ```

## What We've Tried

1. **Fresh cookie export** from Chrome (all YouTube + Google auth cookies) — same error
2. **Swapped Node.js for Deno** in Dockerfile — Node.js can get OOM-killed in Docker containers
3. **Added bgutil-ytdlp-pot-provider** sidecar container — generates PO tokens automatically
4. **Installed bgutil plugin** via pip — bridges yt-dlp to the PO token provider
5. **Configured extractor_args** to point at `http://pot-provider:4416`

## What Still Needs Investigation

- **Check if the plugin is actually being loaded** — run `docker exec corecore-bubble-radio-app-1 yt-dlp --list-plugins` or check verbose output with `-v`
- **Check if the pot-provider is reachable** from the app container — `docker exec corecore-bubble-radio-app-1 curl http://pot-provider:4416`
- **Check detailed yt-dlp error** — the app swallows the full traceback; run a manual test: `docker exec corecore-bubble-radio-app-1 yt-dlp -v --cookies /usr/src/app/cookies/youtube.com.txt --extractor-args "youtube:player_client=default;youtubepot-bgutilhttp:base_url=http://pot-provider:4416" --skip-download "https://www.youtube.com/watch?v=fYT7969YIwU"`
- **Check if Deno is actually available** — `docker exec corecore-bubble-radio-app-1 deno --version`
- **Try without cookies** — PO tokens alone may work for public videos (the error is bot detection, not auth)
- **Check pot-provider logs** — `docker logs corecore-pot-provider-1` to see if token generation requests are arriving and succeeding
- **IP reputation** — datacenter IPs (like the server's) are heavily flagged by YouTube. May need a residential proxy.
- **Memory** — ensure app container has enough memory for Deno JS challenge solving (at least 512MB)

## Other Approaches Not Yet Tried

- **Manual PO token** — extract from YouTube Music network requests, pass via `--extractor-args "youtube:po_token=..."` (expires every ~12hrs, not practical for automation)
- **Residential proxy** — route yt-dlp traffic through a residential IP to avoid datacenter flagging
- **`--force-ipv4`** — some users report success forcing IPv4
- **Different player clients** — try `--extractor-args "youtube:player_client=mweb"` or `tv`
- **Browser-based approach** — use `--cookies-from-browser` (not possible in Docker without a real browser)

## Non-YouTube Workaround

While YouTube is broken, the bot and server work fine with other supported sites (SoundCloud, Bandcamp, etc.). The downloader supports any site yt-dlp supports.
