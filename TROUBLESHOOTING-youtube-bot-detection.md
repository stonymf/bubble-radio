# YouTube "Sign in to confirm you're not a bot" — yt-dlp

## Status: ACTIVE (Mar 2026)

YouTube downloads are failing again. Bandcamp and SoundCloud work fine. The `!testdownloads` bot command confirms YouTube is the only failure.

## Current Problem

Fresh cookies are exported, scp'd to the server, and verified present in the container (27 cookies including `LOGIN_INFO`, `__Secure-1PSID`, etc.) — but yt-dlp still reports "Sign in to confirm you're not a bot."

### What we've tried this session (Mar 13, 2026)

1. **Exported fresh cookies** from Chrome using "Get cookies.txt LOCALLY" extension — full 27-cookie set with all `__Secure-*` variants and `LOGIN_INFO`.

2. **Scp'd cookies to server** at `~/dev/corecore/cookies/youtube.com.txt`. Verified in container via `grep -c`. Cookies are gitignored so they must be scp'd manually.

3. **Fixed `remote_components` API format** — yt-dlp changed the Python API from `{"ejs": "github"}` (dict) to `["ejs:github"]` (list of strings). The old dict format was silently ignored, which meant Deno couldn't download the challenge solver script. Fixed in `src/downloader.py`.

4. **First test succeeded** — ran `download_audio()` directly via `docker exec` and it downloaded "Last Songs (Remastered)" successfully. Logs showed `Downloading tv downgraded player API JSON` and `Downloading challenge solver lib script` — both signs of working cookies + remote_components.

5. **Subsequent tests failed** — yt-dlp had **overwritten the cookie file** during the successful download. The file header changed to "This file is generated by yt-dlp" with only ~19 stripped-down cookies (missing `LOGIN_INFO` and all `__Secure-*` auth cookies). YouTube had rotated the session.

6. **Added `cookies_update=False`** to `_get_ydl_opts()` in `src/downloader.py` to prevent yt-dlp from overwriting the cookie file. This is the `--no-cookies-update` CLI equivalent.

7. **Scp'd cookies again** after the fix was deployed. Verified `LOGIN_INFO` present. But `!testdownloads` still fails with the same "Sign in to confirm you're not a bot" error.

### Remaining hypotheses

- **Cookie invalidation on first use from server IP** — YouTube may have invalidated the session cookies after the first successful download from the datacenter IP (rashomon.blue). The cookies were exported from a residential IP (Chrome on a Mac) and used from a datacenter. YouTube is known to tie sessions to IP ranges.

- **Cookie file format corruption** — the Write tool or scp may have corrupted tab separators in the Netscape cookie format. Verify with `cat -A` to check for proper `\t` characters between fields.

- **yt-dlp cookie caching** — yt-dlp or the gunicorn worker may be caching stale cookie state in memory from before the file was replaced. Try restarting the app container AFTER scp'ing cookies (not before).

- **Pot-provider version mismatch** — logs show `plugin: 1.3.1, HTTP server: 1.2.2`. The bgutil Docker image may need updating to match the pip package version.

## Architecture

```
cookies/youtube.com.txt  (on host, volume-mounted into container)
     ↓
src/downloader.py: _get_ydl_opts()
     → cookiefile = /usr/src/app/cookies/youtube.com.txt
     → cookies_update = False  (new — prevents yt-dlp from overwriting)
     → remote_components = ["ejs:github"]  (new format — was dict, now list)
     → extractor_args.youtube.player_client = ["default"]
     → extractor_args.youtubepot-bgutilhttp.base_url = ["http://pot-provider:4416"]
     ↓
yt-dlp → Deno (JS challenge solver) → YouTube API
```

## Signs It's Working

In verbose output, look for:
- `Found YouTube account cookies` — cookies are being read
- `Downloading tv downgraded player API JSON` — auth is working (NOT `android vr player API`)
- `Downloading challenge solver lib script` — remote components working
- `Downloading 1 format(s): ...` — success

## Signs It's Broken

- `The provided YouTube account cookies are no longer valid` — cookies expired or were overwritten
- `Downloading android vr player API JSON` — falling back because auth failed
- `Remote components challenge solver script (deno) and NPM package (deno) were skipped` — remote_components not being passed correctly
- `Sign in to confirm you're not a bot` — final failure

## Previous Resolution (Feb 2026)

Two issues were fixed:

1. **Incomplete cookie file** — exported cookies were missing critical auth cookies. Needed ~26 cookies including all `__Secure-*` variants.

2. **Missing `remote_components` config** — yt-dlp needs `remote_components` to download JS challenge solver scripts for Deno.

## Things that have changed since Feb fix

- **yt-dlp version:** 2026.02.21 → 2026.03.13
- **remote_components format:** `{"ejs": "github"}` → `["ejs:github"]` (API changed)
- **bgutil plugin:** 1.2.2 → 1.3.1 (but HTTP server Docker image is still 1.2.2)
- **Added `cookies_update=False`** to prevent yt-dlp from overwriting cookie file
- **Added `/test_downloads` endpoint** and `!testdownloads` bot command for testing
